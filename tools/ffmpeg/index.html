ffmpeg

This note is mainly about a famous tool to handle videos and audios. As its official website introduces, ffmpeg is a solution to record, convert and stream audio and video. In my opinion, this tool has many advantages, like open-source, which means no ads, cross-platform, which satisfies various requirements. Thus, I highly recommend this tool to everyone.

Basic Usage

It is known to all that a movie (or sound) file usually consists of video streams, audio streams and subtitles, etc. Each of them is a sequence of frames and has its own encoding and decoding methods. Apart from data conding, sometimes we want to apply some changes to data contents, such as enlarging frame size, changing fps, etc. This explains the fact that the ffmpeg has five inner parts: - demuxer: split the input file(s) into streams - decoder: decode a stream to raw data(frames). - filter: modify data contents - encoder: encode data(frames) to a stream - muxer: combine streams to an output file. As long as you use ffmpeg, your handing process will include some or all of these four parts in order.

ffmpeg calls the libavformat library (containing demuxers) to read input files and get packets containing encoded data from them. When there are multiple input files, ffmpeg tries to keep them synchronized by tracking lowest timestamp on any active input stream.

Encoded packets are then passed to the decoder (unless streamcopy is selected for the stream, see further for a description). The decoder produces uncompressed frames (raw video/PCM audio/...) which can be processed further by filtering (see next section). After filtering, the frames are passed to the encoder, which encodes them and outputs encoded packets. Finally those are passed to the muxer, which writes the encoded packets to the output file.

Terminology: PCM
  Pulse-code modulation (PCM) is a method used to digitally represent sampled analog signals. It is the standard form of digital audio in computers, compact discs, digital telephony and other digital audio applications. In a PCM stream, the amplitude of the analog signal is sampled regularly at uniform intervals, and each sample is quantized to the nearest value within a range of digital steps.
For muxer and demuxer, their implementations are simple. Maybe ffmpeg itself implement them. We just need to give it some arguments.

However, encoder and decoder are different. For example, some rich man want to see high resolution movie but don't care file size, so they need avi format and matching coders, while others care file size more because of storage expense, so mp4, mkv format and lossy compression algorithm like H264, H265 and AAC are suitable for them.
