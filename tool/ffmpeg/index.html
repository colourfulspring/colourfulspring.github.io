    FFmpeg

    本说明主要介绍一个处理视频和音频的著名工具——FFmpeg。正如其官方网站所介绍的，FFmpeg是一个记录、转换和流式传输音频和视频的解决方案。在我看来，这个工具有很多优点，比如开源，没有广告，跨平台，能满足各种常见需求。因此，我向大家高度推荐这个工具。

    基本理解

    众所周知，电影（或声音）文件通常由视频流、音频流和字幕等组成。它们中的每一个都是一系列数据帧，并拥有自己的编码和解码方法。除了数据编码之外，有时我们还想对数据内容进行一些更改，例如放大帧大小、更改fps等。这就解释了FFmpeg有五个内部模块的事实：

    • 解复用器demuxer: 将文件拆成数据流
    • 解码器decoder: 将流解码为原始数据（帧）
    • 过滤器filter: 修改数据内容
    • 编码器encoder: 将原始数据（帧）编码为流
    • 复用器muxer: 将流合并到输出文件

    只要使用FFmpeg，数据的处理过程将按顺序包括这五个模块中的部分或全部。

    文档

    FFmpeg中每个输出的转码过程可以用下图描述：

|       |            |              |
| input |  demuxer   | encoded data |   decoder
| file  | ---------> | packets      | -----+
|_______|            |______________|      |
                                           v
                                       _________
                                      |         |
                                      | decoded |
                                      | frames  |
                                      |_________|
                                           |
                                           |filter
                                           |
                                           v
                                       _________
                                      |         |
                                      | decoded |
                                      | frames  |
                                      |_________|
 ________             ______________       |
|        |           |              |      |
| output | <-------- | encoded data | <----+
| file   |   muxer   | packets      |   encoder
|________|           |______________|

    FFmpeg读取输入文件，首先输入文件进入解复用器，并获取包含编码数据的数据包。当有多个输入文件时，FFmpeg会通过跟踪任何活动输入流上的最低时间戳来保持它们的同步。

    然后编码的数据包被传送到解码器（除非使用copy过滤器），解码器会产生未压缩的数据帧（原始视频/PCM音频等等），并继续送给过滤器。

    在经过过滤器之后，数据帧被传送至编码器，在那里编码为编码数据。

    最后，编码数据被传送给复用器，它将编码数据写入输出文件。

    PCM

    Pulse-code modulation (PCM) is a method used to digitally represent sampled analog signals. It is the standard form of digital audio in computers, compact discs, digital telephony and other digital audio applications. In a PCM stream, the amplitude of the analog signal is sampled regularly at uniform intervals, and each sample is quantized to the nearest value within a range of digital steps.

    对于复用器和解复用器，它们的实现很简单。也许FFmpeg本身就可以实现它们。我们只需要给它们一些参数，指定某个流，它们就可以完成提取流的工作。

    然而，编码器和解码器是不同的。例如，一些富人想看高分辨率的电影，但不关心文件大小，因此他们需要avi格式和匹配的编码器，而其他人则因为存储费用而更关心文件大小，因此mp4、mkv格式和H264、H265和AAC等有损压缩算法适合他们。这些不同的编码器、解码器都要单独实现并安装到电脑中，我们需要显式指定使用哪个编码器。

    过滤器

    在编码之前，FFmpeg可以使用libavfilter库中的过滤器处理原始音频和视频帧。几个链式过滤器构成一个过滤器图。FFmpeg区分两种类型的过滤器图：简单和复杂。

    简单的过滤器图只是相连的过滤器链。前者的输出被视为后者的输入。对于视频和音频，我们分别使用带-vf和-af别名的-filter选项。

    复杂过滤器图不能简单地描述为应用于一个流的过滤器链，而是更复杂的图形，此处暂时不讲。

    crop

    Crop the input video or image.

    Option Description
    -vf crop=w:h:x:y Crop area with size w*h at position (x,y)

    scale

    Resize the input video or image.

    Option Description
    -vf scale=w:h resize the input to a size of w*h

    复用器和解复用器

    如前所述，muxer和demuxer是处理流的工具。为了完成这项工作，提供了一个-map选项和stream specifier。

    Stream Specifier

    Use -map as an example to tell stream specifier

    Option Description
    -map 0:a:1 input file 0, audio stream 1

    编码器和解码器

    为一个或多个流选择编码器（在输出文件之前使用时）或解码器（在输入文件之前使用时）。codec是解码器/编码器或特殊值copy（仅限编码器，即输出）的名称，用于指示流不进行重新编码。 -codec:v，-codec:a或其缩写-c:v，-c:a表示编码器和解码器。

    NVIDIA 支持

    如果您的计算机安装了NVIDIA图形卡，您可以使用cuda加速编码和解码过程。

    使用 ffmpeg -decoders | find "cuvid" (Windows), ffmpeg -decoders | grep cuvid (Linux), 将打印可用NVIDIA解码器的列表。在我的电脑中，结果如下： 

     V..... av1_cuvid            Nvidia CUVID AV1 decoder (codec av1)
 V..... h264_cuvid           Nvidia CUVID H264 decoder (codec h264)
 V..... hevc_cuvid           Nvidia CUVID HEVC decoder (codec hevc)
 V..... mjpeg_cuvid          Nvidia CUVID MJPEG decoder (codec mjpeg)
 V..... mpeg1_cuvid          Nvidia CUVID MPEG1VIDEO decoder (codec mpeg1video)
 V..... mpeg2_cuvid          Nvidia CUVID MPEG2VIDEO decoder (codec mpeg2video)
 V..... mpeg4_cuvid          Nvidia CUVID MPEG4 decoder (codec mpeg4)
 V..... vc1_cuvid            Nvidia CUVID VC1 decoder (codec vc1)
 V..... vp8_cuvid            Nvidia CUVID VP8 decoder (codec vp8)

    使用 ffmpeg -encoders | find "cuvid" (Windows), ffmpeg -encoders | grep cuvid (Linux), 将打印可用NVIDIA编码器的列表。在我的电脑中，结果如下： 

    V....D h264_nvenc           NVIDIA NVENC H.264 encoder (codec h264)
V....D hevc_nvenc           NVIDIA NVENC hevc encoder (codec hevc)

    Copy

    Copy是通过向-codec选项提供copy参数来选择的一种模式。它使FFmpeg省略了指定流的解码和编码步骤，因此它只进行解复用和复用。由于没有解码或编码，因此速度非常快，并且没有质量损失。然而，由于许多因素，它在某些情况下可能不起作用。应用过滤器显然也是不可能的，因为过滤器可以处理未压缩的数据。

    Option Description
    -c:v:1 libx264 use libx264(CPU) to encoded video stream 1
    -c:a copy copy audio stream
    -c:v:1 h264_cuvid use h264_cuvid(GPU) to encoded video stream 1
